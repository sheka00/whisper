# Whisper AI + Speaker Diarization Service

Современный и оптимизированный веб-сервис для распознавания речи (транскрибации) и разделения голосов по ролям (диаризации). Построен на базе FastAPI, **faster-whisper** (движок CTranslate2) и **pyannote.audio**.

## 🚀 Особенности и оптимизации

- **⚡ Невероятная скорость инференса**:
  - Интегрирована пакетная обработка **`BatchedInferencePipeline`** (обработка аудио сегментов параллельными батчами на GPU вместо последовательной генерации).
  - Использование модели `large-v3-turbo` в связке с `compute_type="float16"` и жадным поиском (`beam_size=1`) на видеокартах класса RTX дает скорость до **15x-20x быстрее реального времени** (32 минуты аудио расшифровываются менее чем за 1.5 минуты).
- **👥 Разделение по ролям (Диаризация)**:
  - Автоматическое определение спикеров и сопоставление реплик с помощью предобученной модели `pyannote/speaker-diarization-3.1`.
  - Возможность гибкой настройки: указание точного числа спикеров, минимального или максимального порога участников прямо из веб-интерфейса.
  - Опциональный тумблер включения/выключения диаризации (при выключении транскрибация проходит практически мгновенно, экономя время на стадии кластеризации).
- **🎞️ Универсальная обработка FFmpeg**:
  - Полностью исключены тяжелые Python-библиотеки конвертации видео (`moviepy`, `pydub`), файлы обрабатываются нативным системным процессом `ffmpeg`.
  - Автоматическая поддержка любых форматов: видео (`.mp4`, `.mkv`, `.mov`, `.webm`, `.ts`, `.vob`) и аудио (`.mp3`, `.wav`, `.m4a`, `.ogg`, `.aac`, `.opus`, `.amr`, `.wma` и др.) с конвертацией в оптимальный для моделей формат 16 кГц mono WAV.
- **🧹 Чистота и стабильность**:
  - На бэкенде настроена автоматическая очистка временного кэша `uploads/` при каждом перезапуске контейнера.
  - Временные файлы удаляются с диска сразу после завершения генерации (или в случае возникновения исключений).
- **🎨 Премиальный веб-интерфейс**:
  - Интерактивный UI с красивыми эффектами (стеклянный морфизм, фоновые градиенты, микро-анимации).
  - Стриминг логов в реальном времени через Server-Sent Events (SSE) (пользователь видит каждый шаг: конвертацию, диаризацию, Whisper-распознавание).
  - Копирование текста в один клик и скачивание структурированного диалога в виде файла.

## 🛠️ Подготовка и настройка

1. Скопируйте файл конфигурации:
   ```bash
   cp .env.example .env
   ```
2. Откройте `.env` и вставьте ваш Hugging Face токен в переменную `HF_TOKEN`.
   > **Как получить доступ к модели диаризации:**
   > 1. Примите пользовательские соглашения на Hugging Face для моделей:
   >    - [pyannote/speaker-diarization-3.1](https://huggingface.co/pyannote/speaker-diarization-3.1)
   >    - [pyannote/speaker-diarization-community-1](https://huggingface.co/pyannote/speaker-diarization-community-1) (для xvec_transform.npz)
   > 2. Создайте токен доступа с правами `Read` в [Hugging Face Settings](https://hf.co/settings/tokens) и укажите его в `.env`.

## 🐳 Запуск через Docker Compose

Для запуска на сервере с NVIDIA GPU (требуется установленный [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html)):

```bash
docker compose up -d --build
```

После этого веб-интерфейс станет доступен по адресу:
👉 **[http://localhost:18082](http://localhost:18082)**

*При первом запуске сервис автоматически скачает модель Whisper (~1.6 ГБ) и веса Pyannote. Они кэшируются во внешнем томе Docker, поэтому повторные перезапуски будут мгновенными.*

---

## 🛑 Остановка сервиса

```bash
docker compose down
```
