# Система Локальной Транскрибации (On-Premise B2B)

Комплексное корпоративное решение для безопасного распознавания речи (транскрибации) и разделения голосов по ролям (диаризации). Продукт предназначен для развертывания в закрытом контуре заказчика (on-premise) и гарантирует абсолютную конфиденциальность данных.

Проект состоит из **двух основных модулей**:
1. **Продающий B2B Лендинг** (сбор заявок на демо, описание преимуществ, юридические документы).
2. **Защищенное Web-Приложение** (интерфейс для загрузки файлов и взаимодействия с нейросетями).

---

## 🌟 Ключевые возможности

### 1. Коммерческая оболочка (Лендинг)
- **Конверсионный дизайн**: Современный премиальный UI с микро-анимациями, адаптивный под все мобильные устройства.
- **Генерация лидов**: Форма «Запросить демо» сохраняет заявки (Имя, Email, Телефон, Компания) в локальный файл `uploads/leads.json` для последующей обработки менеджерами.
- **Юридическая база**: Встроены страницы «Политика конфиденциальности» и «Пользовательское соглашение», адаптированные под требования 152-ФЗ для on-premise продуктов.

### 2. Ядро обработки (Нейросети)
- **Молниеносный инференс**: Базируется на **faster-whisper** (CTranslate2) с пакетной обработкой (`BatchedInferencePipeline`). На видеокартах класса RTX скорость распознавания достигает **15x-20x быстрее реального времени**.
- **Диаризация**: Автоматическое определение спикеров с помощью `pyannote.audio` с возможностью тонкой настройки количества участников.
- **Универсальность**: Нативная конвертация любых видео и аудио форматов (`.mp4`, `.mkv`, `.mp3`, `.wav` и др.) через системный `ffmpeg` без перегрузки Python-памяти.
- **Очередь задач**: Встроенная система очередей для защиты GPU от переполнения памяти (OOM) при одновременной загрузке нескольких тяжелых файлов.

## 🛠️ Установка и запуск

### 1. Подготовка окружения
Скопируйте файл конфигурации и настройте его:
```bash
cp .env.example .env
```

Внутри `.env` необходимо указать:
- `HF_TOKEN` — ваш токен Hugging Face (требуется для доступа к моделям `pyannote/speaker-diarization-3.1`).
- Учетные данные для доступа к самому приложению (`ACCESS_USERNAME`, `ACCESS_PASSWORD`).

### 2. Запуск через Docker Compose
Для работы нейросетей требуется сервер с NVIDIA GPU и установленным [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html).

```bash
docker compose up -d --build
```

### 3. Структура приложения
После запуска сервис будет доступен по адресу `http://localhost:18082`:
*   `/` — Главная страница (Лендинг).
*   `/app` — Рабочий интерфейс транскрибации (закрыт паролем).
*   `/privacy` — Политика конфиденциальности.
*   `/terms` — Пользовательское соглашение.

Заявки с сайта сохраняются в директорию `uploads/`, которая проброшена на хост-машину (внешний диск сервера), поэтому они не удаляются при перезапуске контейнера.
